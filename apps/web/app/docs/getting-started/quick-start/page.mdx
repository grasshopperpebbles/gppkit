# Quick Start

This guide will walk you through creating a multi-platform project with GPP.

## Create a Project

```bash
gpp init my-project
cd my-project
```

This creates the GPP project structure:

```
my-project/
├── .gpp/           # GPP configuration
├── apps/           # Platform applications
├── docs/           # Project documentation
└── logs/           # Development logs
```

## Add a Web Platform

```bash
gpp add web
```

This scaffolds a Next.js application in `apps/web/`.

### Available Web Variants

- `webapp` - Generic Next.js app (default)
- `saas-dashboard` - SaaS dashboard with auth, billing, user management
- `admin-dashboard` - Admin panel with data tables
- `marketing` - Marketing site with landing pages
- `landing` - Simple landing page
- `devtool-site` - Documentation site (like this one!)
- `headless-strapi` - Next.js frontend for Strapi CMS
- `headless-wp-cms` - Next.js frontend for WordPress
- `headless-wp-ecommerce` - Next.js + WooCommerce

Specify a variant:

```bash
gpp add web saas-dashboard
```

## Add an API Platform

```bash
gpp add api express
```

This creates an Express.js API in `apps/api/`.

### Available API Variants

- `express` - Node.js Express API

> **Note**: For Python APIs, use `gpp add python fastapi` instead.

## Add a CMS Platform

```bash
gpp add cms strapi-only
```

This creates a Strapi CMS in `apps/cms/`.

### Available CMS Variants

- `strapi-only` - Strapi 5 CMS (production)
- `strapi-dev` - Strapi 5 CMS (development)
- `wp-content` - WordPress with Docker
- `cms-only` - Basic CMS setup

## Add a Mobile Platform

```bash
gpp add mobile expo
```

### Available Mobile Variants

- `expo` - React Native with Expo Router
- `flutter-standalone` - Flutter standalone app
- `flutter-api-client` - Flutter with API client integration

## Start Development

Start any platform from the root:

```bash
# Start web
gpp start web

# Start API
gpp start api

# Start CMS
gpp start cms

# Start mobile
gpp start mobile
```

Or navigate to the platform directory:

```bash
cd apps/web
gpp start
```

GPP auto-detects the current platform.

## Install Dependencies

GPP creates the project structure but doesn't install dependencies automatically. After adding a platform:

```bash
cd apps/web
npm install
```
