# CMS Platform

The CMS platform provides content management capabilities using Strapi or WordPress.

## Variants

### strapi-only

A Strapi 5 CMS for production use.

```bash
gpp add cms strapi-only
```

When you add the CMS, GPP creates `.env` and the web app’s `.env.local` so the CMS is ready to run. A first-start script runs automatically in the container (e.g. installs the GraphQL plugin), so **`gpp start cms`** or **`docker compose up -d`** is enough — no separate setup step.

**Includes:**
- Strapi 5
- MySQL database (Docker)
- GraphQL plugin (installed on first container start)
- SEO plugin
- CKEditor 5

### strapi-dev

A Strapi 5 CMS configured for development and plugin testing.

```bash
gpp add cms strapi-dev
```

Same env-on-add and first-start behavior as strapi-only; uses separate ports (e.g. 1338) for isolation.

### WordPress Variants

WordPress headless CMS with Docker (MariaDB + Redis + phpMyAdmin).

```bash
gpp cms setup
```

| Variant | Description |
|---------|-------------|
| `cms-only` | WordPress headless CMS without WooCommerce |
| `cms-ecommerce` | WordPress with CMS and WooCommerce (blog primary) |
| `ecommerce-only` | WordPress with WooCommerce only |
| `ecommerce-cms` | WordPress with CMS and WooCommerce (shop primary) |

**Includes:**
- WordPress latest with Redis PHP extension
- MariaDB 11, Redis 7, phpMyAdmin
- Auto-install of standard plugins (WPGraphQL, ACF, Redis Cache, Rank Math, JWT Auth)
- Five bundled custom plugins (gp-featured-items, gp-headless-config, gp-hero-slider, gp-most-popular, gp-social-poster)
- Hero slider for rotating homepage images (ACF + WPGraphQL)

See [gpp cms](/docs/commands/cms) for full setup documentation.

## Configuration

CMS path and scripts are in `.gpp/deploy.json` (e.g. `platforms.cms.path` like `apps/<project>-node-strapi-cms`). GPP uses this path for `gpp start cms`, `gpp strapi --setup`, and production so dev and prod stay in sync.

```json
{
  "platforms": {
    "cms": {
      "path": "apps/myproject-node-strapi-cms",
      "variant": "strapi-only",
      "scripts": {
        "start": "npm run develop",
        "build": "npm run build"
      }
    }
  }
}
```

## Commands

```bash
# Start CMS
gpp start cms

# Build for production
gpp build cms
```

## Connecting to Web

Use `gpp wire` to connect your CMS to a web frontend:

```bash
gpp wire cms web
```

This generates:
- API client for fetching content
- Type definitions for content types
- Environment variable configuration

## WordPress Development

```bash
cd apps/cms
docker-compose up -d
```

Access:
- WordPress: http://localhost:8080
- phpMyAdmin: http://localhost:8081

## Strapi Development

Start the CMS (path from `.gpp/deploy.json`); first-start runs automatically in the container once:

```bash
gpp start cms
```

Or from the CMS directory:

```bash
cd apps/<your-cms-path>   # e.g. apps/myproject-node-strapi-cms
docker compose --env-file .env up -d
```

Access:
- Strapi Admin: http://localhost:1337/admin (or 1338 for strapi-dev)
- API: http://localhost:1337/api
- GraphQL: http://localhost:1337/graphql

## Strapi 5 Plugins

The Strapi templates include these Strapi 5 compatible plugins:

- `@strapi/plugin-graphql` - GraphQL API
- `@strapi/plugin-documentation` - API documentation
- `@strapi/plugin-seo` - SEO management
- `@_sh/strapi-plugin-ckeditor` - Rich text editor
- `strapi-plugin-navigation` - Navigation management
- `strapi-plugin-comments` - Comments system
- `strapi-plugin-config-sync` - Config synchronization
