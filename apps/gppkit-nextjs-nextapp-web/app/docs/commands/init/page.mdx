# gpp init

Initialize a new GPP project with the standard directory structure.

## Usage

```bash
gpp init <project-name> [options]
```

## Arguments

| Argument | Description | Required |
|----------|-------------|----------|
| `project-name` | Name of the project directory to create | Yes |

## Options

| Option | Description |
|--------|-------------|
| `--recipe <id>` | Use a specific recipe (e.g., `nextjs-strapi`, `flutter-fastapi`) |
| `--with-wizard` | Use guided wizard for non-technical users |
| `--dry-run` | Generate a plan without executing |

## Interactive Mode (Default)

When you run `gpp init` without the `--recipe` flag, it launches an interactive recipe selector:

```bash
gpp init my-saas
```

You'll be prompted to choose:
1. **Project category** - E-commerce, Headless CMS, SaaS Dashboard, Mobile App, Landing Page
2. **Tech stack** - Specific technologies within that category

This is the recommended approach for new projects.

## Direct Recipe Mode

Skip the interactive prompts by specifying a recipe directly:

```bash
# SaaS with Go backend
gpp init my-saas --recipe nextjs-go-postgres
gpp init my-saas --recipe sveltekit-go-postgres

# SaaS with Python backend
gpp init my-saas --recipe nextjs-fastapi-postgres

# Headless CMS
gpp init my-blog --recipe nextjs-strapi
gpp init my-blog --recipe nextjs-sanity

# Mobile app
gpp init my-app --recipe flutter-fastapi
```

## Available Recipes

### SaaS Dashboard
| Recipe | Frontend | Backend |
|--------|----------|---------|
| `nextjs-go-postgres` | Next.js | Go HTTP |
| `sveltekit-go-postgres` | SvelteKit | Go HTTP |
| `nextjs-fastapi-postgres` | Next.js | FastAPI |
| `nextjs-supabase` | Next.js | Supabase |

### Headless CMS
| Recipe | Frontend | CMS |
|--------|----------|-----|
| `nextjs-strapi` | Next.js | Strapi |
| `nextjs-sanity` | Next.js | Sanity |
| `nextjs-storyblok` | Next.js | Storyblok |

|--------|-----------|---------|

## Guided Wizard

For non-technical users, the wizard asks intent-based questions:

```bash
gpp init my-project --with-wizard
```

The wizard guides you through questions like "What type of project are you building?" rather than asking about specific technologies.

## What It Creates

```
project-name/
├── .gpp/
│   ├── config.json      # Project configuration
│   ├── deploy.json      # Platform settings
│   └── plans/           # Execution plans
├── apps/                # Platform applications
├── docs/                # Project documentation
└── logs/                # Development logs
```

## Examples

### Interactive recipe selection

```bash
gpp init my-saas
# → Select category: SaaS Dashboard
# → Select stack: Next.js + FastAPI + PostgreSQL
```

### Direct recipe for CI/scripting

```bash
gpp init my-blog --recipe nextjs-strapi
cd my-blog
gpp start
```

### Preview what would be created

```bash
gpp init my-project --recipe flutter-fastapi --dry-run
```

## Configuration Files

### .gpp/config.json

Stores project-level configuration:

```json
{
  "name": "my-saas",
  "version": "0.1.0"
}
```

### .gpp/deploy.json

Stores platform configurations:

```json
{
  "platforms": {}
}
```

## Next Steps

After initializing, you can add more platforms or start development:

```bash
# Start all platforms
gpp start

# Or add additional platforms
gpp add web saas-dashboard
gpp add api express
gpp add cms strapi-only
```
